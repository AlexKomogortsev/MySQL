Данная база используется для отображение статитики бойцов ММА, на основании которой будет создан сайт, отображающий всю необходимую информацию для денежных ставок в данном спорте.
Следует отметить, что бойцы, выступающие в лиге UFC (самая крупная и известная организация в мире) имеют более "глубокую" статистику, что делает их интереснее для букмейкеров и аналитиков. В связи с этим мы разделили общую информацию о ВСЕХ бойцах и дополнительную о тех, кто выступает непосредственно в лиге UFC.
Данные взяты рэндомные с помощью сайта filldb и мы старались генерировать их близко к реальным.

1) Таблица ufc_event: отображает общую информацию о турнирах компании UFC. Связана с таблицами fight_all и ufc_fight. 
2) Таблица fight_all: отображает статистику для всех бойцов. Бой может быть планируемым и прошедшим - для этого есть столбец статус и поэтому же есть fighter1/figther2 + winner/loser - если бой планируется, то мы ещё не можем определить победителя или проигравшего. Таблица связана с ufc_fight по id бойца, fighter_all для колонок fighter1 и fighter2 по id бойца, ufc_event по id события, если таковой есть.
В следующей версии планируется сократить количество связей, например, убрать из этой таблицы ufc_event_id и оставить его только в таблице ufc_fight
3) Таблица ufc_fight: отображает дополнительную статистику боёв, прошедших под эгидой ufc; а именно количество ударов, переводов на землю и т.п для каждого бойца. В жизни на основании данной таблицы формируется статистика таблицы ufc_fighter и букмейкерские коэффициенты таблицы possible_outcome. Но конкретно в нашем случае, берём для тех таблиц уже готовые данные с других сайтов, и связываем их по id бойца и по id боя. Также таблица имеет id турнира.
4) Таблица possible_outcomes отображает коэффициенты букмейкеров на бой UFC, которые парстся с релевантных сайтов.
5) Таблица fighter_all: содержит общую информацию о бойце.
6) Таблица ufc_fighter: если боец дрался в ufc, то у него есть своя статистика, сведенная по проведенным схваткам. Таблица тянет id бойца из fighter_all и передает его в таблицу ufc_fight.

Для нашей базы формировалось два сложных select запроса, один их которых заключен в процедуру: 
Первый - отображает список ufc турниров, имеющих самый высокий индекс нокаутов (или самых нокаутирующих бойцов); используем inner join'ы, таблицы-представления, одна из которых содержит union.
Второй - отображает самого "крутого" бойца. Таблица вытаскивает из разных мест имя бойцов, количества их побед и поражений. Используются временные таблицы, outer join'ы. Также данный запрос заключен в процедуру.

В корне есть скрипт с формированией базы и наполнением огромным массивом тестовы данных. Используем параметр SET FOREIGN_KEY_CHECKS
Также сформирована диаграмма.
